# Git branch and ticket

Create or align a **feature branch** from **`release`** (or team base) and tie work to a **POST-*** ticket.

Follow: **`.cursor/docs/workflows/tasks/implement-feature/01-git-branch-and-ticket.md`**.

Branch naming and Jira: **`.cursor/rules/jira-ticket-management.mdc`**.

**Frontend `test-update`:** once you implement **`frontend/`** changes (**`.cursor/rules/task-verification.mdc`**), run **`npm run test-update`** from **`frontend/`** before committing.
