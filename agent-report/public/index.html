Send a report

Run this from the app folder. Use --automation-name for the automation, --title for the headline, --status for the colored chip, and the quoted text as the message.

Status examples: success, warning, error, running, pending, info.

Direct WebSocket JSON can send automationName, title, status, and text.

Automation reporting contract

Tell every automation to report start, major step completion, blockers, and final state to this page.

Always send automationName, title, status, and text. Keep source as terminal or browser.

When any item finishes, report it immediately before moving on: ticket, PR, vulnerability, listing, persona review, file sync, subagent assignment, scan row, or other discrete work item.

At startup, run AGENT_REPORT_SEND_STATUS=0 scripts/ensure-agent-report-server.sh, then send the automation's own start message.

cd /Users/thien.nguyen/thiennp.github.io/agent-report && AGENT_REPORT_WS=ws://localhost:3100/stream npm run send -- --automation-name "<automation name>" --title "<step title>" --status <running|success|warning|blocked|error|pending|info> "<concise update>"
