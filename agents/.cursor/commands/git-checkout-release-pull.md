# Git checkout release and pull

Align the workspace with **latest `origin/release`**: check out local **`release`**, then fast-forward pull from **origin**.

Resolve any pull conflicts before continuing other work.

From **repository root**, use your Git integration to check out **`release`** and update it from **origin** (same intent as **`git checkout release`** then **`git pull origin release`**).

Orientation: **`.cursor/rules/git-workflow.mdc`**.
