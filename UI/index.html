Codex automation

Sentry Cursor Agent

A lightweight monitor for the Sentry feed that hands recent EnergyCenter webview regressions to Cursor with the repository's own bug-fix prompt.

Agent Flow

  1. 1
    Read the authenticated Sentry feed

    Use the active Chrome session, refresh the 24-hour issue feed, and copy only the visible issue list needed for triage.

  2. 2
    Gate on the EnergyCenter project

    Continue only when the latest issue belongs to the EnergyCenter reverse project; otherwise record the latest non-matching issue and stop.

  3. 3
    Open Cursor on the local repository

    Resolve enrg-energycenter-rev, inspect .cursor guidance, and pass Cursor the Sentry context.

  4. 4
    Use the project bug-fix prompt

    Ask Cursor to adapt .cursor/prompts/fix-bug.md, implement a minimal defensive fix, and report changed files plus verification results.

Current Operating Rules

Schedule

Every 5 minutes in the current Codex thread.

Credentials

No passwords, OTPs, passkey secrets, tokens, or raw cookies are copied into prompts or committed.

Write boundary

Cursor may edit the local EnergyCenter repository, but it must not commit, push, or create a PR unless explicitly requested.

Cursor Prompt Template

Use .cursor/prompts/fix-bug.md, adapted for a Sentry issue without a Jira ticket.

Sentry context:
- Project: <latest EnergyCenter issue key>
- Error: <message>
- Route: <route or source shown in feed>
- Last seen: <relative timestamp>
- Impact: <events/users/priority from feed>

Follow the local .cursor rules, including the RAG gate. Investigate root cause
and implement the minimal defensive fix. Update types/tests as needed. Do not
paste secrets. Do not commit, push, or open a PR. Report files changed and
verification commands/results.

This page is intentionally generic so the public repository does not expose private Sentry sessions, credentials, or internal incident payloads.
